Technical Requirements: Sensory Hardware Interaction Haptic Bodysuit

1. Overview

The system enables haptic feedback interactions between a Performer's vest and a Member's vest, mediated via a mobile application, to ensure secure, authorized, and real-time social-haptic experiences.

2. Revised Connection Flow

The architecture relies on an App-to-Cloud-to-Vest handshake to maintain security and low latency.

  • Discovery: Performer's vest/app broadcasts availability via BLE Advertising.
  • Authorization: Upon Member's acceptance, the app sends a secure token to the backend server.
  • Confirmation: Upon server verification, the system triggers an audio signal to both parties.
  • The Trigger: Performer interacts with their vest; the app routes an encrypted packet to the Member's vest.

3. Hardware/Firmware Requirements

  • Controller: Low-power Bluetooth SoC (e.g., nRF52/nRF54 series).
  • Actuators: Linear Resonant Actuators (LRAs) for high-fidelity haptic feedback.
  • Latency Target: < 100ms.
  • Security: Encrypted command packets to prevent spoofing.

5. System Component Categorization

Haptic Bodysuit: Sensory Immersion - Embedded Conductive Traces / LRAs

Mechanical Stimulation: Physical Stimulation - Closed-Loop Linear Actuation

5.1-5.4 Mechanical Integration & Actuation

Key Objectives

  • Actuation: High-torque linear actuators or micro-servo arrays.
  • Latency: < 50ms propagation delay.
  • Safety: Medical-grade, hypoallergenic, water-resistant materials with managed heat dissipation.
  • Closed-Loop: Encoder-based position feedback to ensure input/output synchronization.

6. Interactive Mechanical Stimulation System

6.1 Primary/Secondary Architecture

Primary Controller (Input): Captures spatial data, thrust depth, velocity, and force.

Secondary Actuator (Output): Interprets telemetry and performs corresponding physical displacement.
